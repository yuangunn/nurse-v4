# NurseScheduler v4 — 사용 매뉴얼

## 1. 소개

**NurseScheduler**는 간호사 3교대 근무표를 자동으로 생성하는 데스크톱 앱입니다.

- 수리최적화(PuLP + HiGHS) 솔버 기반으로 최적의 근무표를 계산합니다.
- 인터넷 연결이 필요 없는 완전한 로컬 실행 앱입니다.
- 프로필별 데이터베이스 분리 + Fernet 암호화로 간호사 정보를 보호합니다.

3. 바탕화면 또는 시작메뉴의 **NurseScheduler** 바로가기 클릭

2. 임의 폴더에 압축 해제

### 다운로드

### 시스템 요구사항

## 3. 퀵가이드 — 5단계로 근무표 만들기

### STEP 1: 설정 (첫 1회만)

1. **간호사 등록**: + 추가 버튼으로 개별 등록, 또는 CSV 일괄 등록
   - 📥 템플릿 다운로드 → 엑셀에서 편집 → 📤 가져오기
   - 드래그로 순서 변경 = 시니어리티 (위 = 선임, 차지 우선)
2. **요일별 필요 인원**: D/E/N 인원 설정 (자동배정 근무는 행 자동 추가)
3. **규칙 관리**: 하드 제약 (연속 근무, 전환 금지 등) + 소프트 배점

### STEP 2: 사전입력

1. **주휴(주)** → **연차(V)** → **생리휴가(생)** → **희망 근무** 순으로 입력
2. **셀 클릭** → 근무 선택 팝업, 또는 **키보드 직접 입력** (D/E/N/V/O)
3. **방향키**로 셀 이동, **Del**로 삭제, **Ctrl+Z**로 되돌리기
4. **드래그**로 여러 셀 선택 후 한 번에 채우기 가능
5. 빈 셀은 솔버가 자동으로 채웁니다

### STEP 3: 분석

1. **분석 실행** → 일자별 D/E/N 인원 과부족 확인
2. **주휴 추천 배분** → "사전입력에 적용" 클릭으로 한 번에 반영
3. 경고 확인: 인원 부족/빡빡한 날짜 미리 파악

### STEP 4: 스케줄 생성

1. **"생성" 버튼** 클릭 → 솔버가 최적 근무표 자동 생성
2. 오차(MIP Gap): 정밀(0%) ~ 10% 조절 가능 (높을수록 빠름)
3. 생성 실패 시: **사전입력 완화** → **주휴 무시** → **V 무제한** 순서로 체크
4. 완성된 근무표는 셀 클릭으로 수동 수정 가능 (빨간 테두리 표시)

### STEP 5: 저장 & 내보내기

1. **저장** 버튼으로 DB에 보관 (생성 완료 시 자동 저장)
2. **CSV** 버튼으로 엑셀 호환 파일 내보내기
3. **인쇄** 버튼으로 A3/A4 인쇄

## 4. 주요 기능 상세

### 4.1 프로필 시스템
- 병동별 프로필 생성 (예: 101병동, 102병동)
- 프로필별 별도 DB (간호사/스케줄/규칙 분리)
- 비밀번호 설정 시 Fernet 암호화 적용
- 게스트 모드: 비밀번호 없이 즉시 사용, 종료 시 데이터 삭제

### 4.2 간호사 관리
- 그룹(A/B/C), 성별, 가능 근무 설정
- 야간전담 간호사: 월별 ON/OFF 토글
- 트레이닝 간호사: 프리셉터 연동, 트레이닝 기간 설정
- 전입/전출 로테이션: 날짜 범위 설정, 범위 밖은 자동 제외
- CSV 템플릿 다운로드 / 내보내기 / 가져오기 (14개 필드)

### 4.3 근무 유형
| 코드 | 이름 | 시간 | 배정 |
|------|------|------|------|
| DC | Day Charge | 06:00~14:00 | 자동 |
| D | Day | 06:00~14:00 | 자동 |
| D1 | Day1 (상근) | 08:30~17:30 | 수동 |
| EC | Evening Charge | 14:00~22:00 | 자동 |
| E | Evening | 14:00~22:00 | 자동 |
| 중 | 중간번 | 11:00~19:00 | 설정 가능 |
| NC | Night Charge | 22:00~06:00 | 자동 |
| N | Night | 22:00~06:00 | 자동 |
| OF | Off | — | 자동 |
| 주 | 주휴 | — | 수동 |
| V | 연차 | — | 수동 |
| 생 | 생리휴가 | — | 수동 |

### 4.4 근무 전환 규칙
허용되는 전환 (다음날):
- D → E, N, 중, 휴무 (순방향)
- E → N, 휴무
- 중 → E, N, 중, 휴무
- N → 휴무 (연속야간 후 2일 휴무 규칙 적용 가능)

금지되는 전환:
- E → D, D1, 중 (역순)
- N → D, D1, E, 중 (역순)
- 중 → D, D1 (시간 부족)

### 4.5 사전입력 완화
생성 실패 시 "사전입력 완화"를 체크하면 사전입력한 근무를 솔버가 변경할 수 있습니다.

종류별 차등 보너스:
| 종류 | 기본 보너스 | 설명 |
|------|:---------:|------|
| 휴가 (V/생/특/공/법/병) | 5000 | 간호사 요청 — 거의 유지됨 |
| 근무 (D/E/N/중 등) | 500 | 교체 가능 — 유연하게 조정 |
| 쉬는 날 (OF/주) | 300 | 가장 유연하게 교체 |

스케줄 탭의 "완화 설정"에서 슬라이더로 조절 가능합니다.

### 4.6 분석
- 일자별 D/E/N(+추가 근무) 필요 인원 히트맵
- 여유 인원 색상 표시 (여유/양호/적정/빡빡/부족)
- 주휴 추천 배분 (4주 순환 + 요일별 여유도 기반)
- 간호사별 요약 (D/E/N/휴무/주말/점수)
- 생성 결과 자동 갱신

### 4.7 스케줄 비교
스케줄 탭 "🔀 비교" 버튼으로 현재 vs 저장된 스케줄의 셀 차이를 확인할 수 있습니다.

### 4.8 규칙 프리셋
"엄격" / "유연" 등 규칙 세트를 이름으로 저장하고 한 번의 클릭으로 교체할 수 있습니다.

## 6. 자주 묻는 질문 (FAQ)

**Q: 근무표 생성이 실패합니다 (Infeasible)**
1. 사전입력 완화 체크 → 재시도
2. 실패 시 주휴 무시 추가
3. 실패 시 V 무제한 추가
4. 에러 메시지의 원인 확인 → 해당 규칙 완화

**Q: 생성 시간이 너무 오래 걸립니다**
오차를 2%~5%로 올리면 훨씬 빠르게 생성됩니다.

**Q: 차지(Charge) 순서는 어떻게 결정되나요?**
간호사 목록 순서(위=선임, 아래=후임)가 시니어리티입니다. 드래그로 순서를 변경하세요.

**Q: 다른 PC로 데이터를 옮기려면?**
개발자 설정 → DB 백업 다운로드, 또는 간호사 내보내기(CSV) 사용.

**Q: 데이터는 어디에 저장되나요?**
`%APPDATA%\NurseScheduler\` 폴더에 저장됩니다. 앱을 삭제해도 데이터는 보존됩니다.

## 7. 개발자 모드

사이드바 `v4` 텍스트를 **5회 빠르게 클릭**하면 개발자 모드가 활성화됩니다.

개발자 설정 (사이드바 하단 "개발자" 버튼):
- 마스터 비밀번호 설정/제거
- 프로필 비밀번호 강제 초기화
- localStorage 전체 삭제
- DB 백업 다운로드
- 예시 데이터 초기화
- DB 경로/크기 확인
- 글꼴 범위 확장 (8~30px)
